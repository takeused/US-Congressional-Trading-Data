## 🗺️ 0. 전체 파이프라인 한눈에

```
① 인덱스 다운로드 (연도별 ZIP → XML)
        ↓  FilingType == 'P' (=PTR) 만 필터
② 개별 PDF 다운로드 (DocID 기준)
        ↓  pdftotext -layout 로 텍스트 추출
③ 정규식 파싱 (거래유형 · 날짜 · 금액 · 티커)
        ↓  엣지케이스 처리 (멀티라인 금액 · 스캔 PDF · 필러 오타)
④ 구조화 JSON  {filer, ticker, type, amount, transaction_date, notification_date}
        ↓  yfinance / 브로커 API 로 가격·섹터 인리치
⑤ 대시보드 (수익률 · 수급 · 테마 · 의원 프로필)
```

## 📥 1. 인덱스 — 연도별 공시 목록

하원은 연도별 **전체 공시 인덱스**를 ZIP(내부 XML)으로 배포한다.

```
https://disclosures-clerk.house.gov/public_disc/financial-pdfs/{YEAR}FD.zip
  └─ {YEAR}FD.xml   (예: 2026FD.xml)
```

### XML 핵심 필드 (`<Member>` 항목)

| 필드 | 의미 |
| --- | --- |
| `FilingType` | **`P` = PTR** (개별 거래 공시) · `O`/`A`/`C` 등은 연차보고서 |
| `DocID` | PDF 파일 식별자 (다운로드 URL에 사용) |
| `FilingDate` | 공시 접수일 |
| `Last` / `First` | 의원 이름 |
| `StateDst` | 주/선거구 (예: NJ05) |

```python
import io, zipfile, urllib.request, xml.etree.ElementTree as ET

UA = {"User-Agent": "Mozilla/5.0 ... Chrome/120 Safari/537.36"}  # 봇 차단 회피용 정상 UA

data = urllib.request.urlopen(urllib.request.Request(
    "https://disclosures-clerk.house.gov/public_disc/financial-pdfs/2026FD.zip",
    headers=UA)).read()
xml = zipfile.ZipFile(io.BytesIO(data)).read("2026FD.xml")
root = ET.fromstring(xml)
ptrs = [m for m in root.findall("Member") if (m.findtext("FilingType") or "") == "P"]
```

## 🧩 4. 정규식 파싱 — 거래 한 줄

레이아웃 추출 기준 PTR 거래행 형태:

```
  (자산명) (TICKER) [ST]   P   01/15/2026   01/21/2026   $1,001 - $15,000
   ────────자산────────    ▲       ▲            ▲              ▲
                          유형   거래일        공시일          금액구간
```

```python
import re

# 유형(S매도 / P매수 / E교환) + 거래일 + 공시일 + 금액구간
TXN_RE = re.compile(
    r'([SPE])\s+(\d{2}/\d{2}/\d{4})\s+(\d{2}/\d{2}/\d{4})\s+'
    r'(\$[\d,]+(?:\.\d+)?(?:\s*-\s*\$[\d,]+)?)')

# 티커 + 자산유형([ST]주식 / [OP]옵션 등)
TICKER_RE = re.compile(r'\(([A-Z][A-Z0-9.\-]{0,6})\)\s*\[([A-Z]{2})\]')

typ, tdate, ndate, amt = TXN_RE.search(line).groups()
kind = {"S": "sell", "P": "buy", "E": "exchange"}[typ]
```

### 자산유형 코드

| 코드 | 의미 |
| --- | --- |
| `[ST]` | 주식 |
| `[OP]` | 옵션 |
| `[MF]` | 뮤추얼펀드 |
| `[GS]` | 국채 |

> 대시보드는 보통 `[ST]`만 수익률 계산에 사용 (옵션은 레버리지라 별도 처리).

## ⚠️ 5. 엣지케이스 (실전에서 반드시 만남)

| 문제 | 증상 | 해결 |
| --- | --- | --- |
| **금액구간 줄바꿈** | `$250,001 -` 만 잡히고 상한이 다음 줄 | 다음 1–2줄에서 `$숫자` 재검색해 결합 |
| **스캔 PDF** | `pdftotext` 결과 거의 빈 문자열 | OCR(Tesseract) 또는 LLM 비전 fallback |
| **티커 없음** | 미 국채·비상장·뮤추얼펀드 (~35%) | 티커=null 저장 (테마 차트 등에선 제외) |
| **필러 오타** | 거래일이 공시일보다 미래 | `transaction_date > notification_date` 행 제외 |
| **이름 매칭** | "Hon. Josh Gottheimer" ↔ legislators DB | first+last 정규화, last-name fallback |

```python
# 금액구간 멀티라인 결합 예시
if amt.count("$") < 2 and amt.rstrip().endswith("-"):
    for j in range(i + 1, min(i + 3, len(lines))):
        um = re.search(r'\$[\d,]+(?:\.\d+)?', lines[j])
        if um:
            amt = amt.rstrip(" -") + " - " + um.group(0)
            break
```

## 💹 6. 인리치 — 가격·섹터·기업정보

파싱된 티커에 시세를 붙여 **"따라 샀으면 수익률"** 을 계산한다.

- **가격 이력** — yfinance `history(period="2y")` (오래된 매수도 진입가 계산 가능). 자체 DB(`us_daily_prices`) 폴백
- **현재가 / 누락종목** — 브로커 API(예: 토스증권 Open API) 라이브 시세로 yfinance가 놓친 종목(BRK.B 등) 복구
- **섹터·기업설명** — yfinance `.info` (`sector`, `industry`, `longBusinessSummary`)
- **의원 프로필** — `congress-legislators` (unitedstates.github.io) — 정당·주·상임위 (내부정보 접근 관점)

> 📐 **수익률 정의** — 공시일(=일반인이 따라 살 수 있는 최초 시점) 종가 진입 → 현재가. 표본수(n)·승률 동반 표기.
>

## 🧭 7. 통계 정직성 (중요)

- 모든 수익률에 **표본수 n** 병기 — 소표본(n<5)은 상위 노출 제외
- "과거 성과가 미래를 보장하지 않음" 고지
- 정보 제공용 — **투자 권유 아님**

> ⚠️ 의원 매매 추종의 실증 알파는 대체로 약하고 표본편향이 크다. 대시보드는 **재미·투명성** 도구이지 자동매매 신호가 아니다.
